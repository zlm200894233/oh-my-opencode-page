多智能体协同编码系统

将 AI 编码从单一对话升级为智能体编排 —— 每个智能体各司其职，协同交付工程级代码

系统概览

Oh My OpenCode 是 OpenCode CLI 的插件，将单一 AI 助手升级为多智能体协作系统。 每个智能体都有专门的角色、专属的 AI 模型和明确的职责边界。通过智能编排实现： 意图识别 → 任务分解 → 并行执行 → 质量验证。

灵感来源：西西弗斯每天推石上山，工程师每天推代码上线 —— 我们没什么不同。

工作流程

0
意图识别
  • 研究型："解释X" → 探索 → 综合回答
  • 实现型："实现X" → 规划 → 委派执行
  • 调查型："看看X" → 探索 → 报告发现
  • 评估型："你怎么看" → 评估 → 等待确认
  • 修复型："出错了" → 诊断 → 最小修复
  • 开放型："重构" → 评估 → 提方案
1
代码库评估
  • 规范型 → 严格遵循现有风格
  • 过渡型 → 询问遵循哪种模式
  • 混乱型 → 提出新规范建议
  • 全新型 → 应用现代最佳实践
2
探索与执行
  • 2A 研究：并行发射 Explore + Librarian
  • 2B 实现：通过 Category + Skill 委派
  • 2C 恢复：3次失败 → 停止 → 回滚 → 咨询 Oracle
3
完成验证
  • 所有 TODO 项完成 ✓
  • LSP 诊断检查通过 ✓
  • 构建/测试通过 ✓
  • 用户原始需求满足 ✓

并发控制

通过精细的并发控制，系统在成本、速度和质量之间取得最优平衡。昂贵的推理模型限制并发以控制成本，轻量的搜索模型则允许大量并行以加速研究。

协作示例

一个真实请求如何流经整个多智能体系统

"给我的 Express API 添加 JWT 认证"

01
Sisyphus 接收请求
识别为"实现型"意图，需要规划 → 委派执行流程
02
并行研究启动
同时发射 Explore（搜索现有代码结构）和 Librarian（查找 JWT 安全最佳实践）
03
Prometheus 制定计划
收集研究结果后，创建并行任务图：识别 3 个可独立执行的子任务
04
Momus 审查计划
验证计划的清晰度和完整性，确认无遗漏的边界情况
05
并行执行
deep → JWT 中间件 ｜ deep → 登录/注册接口 ｜ quick → 配置文件更新
06
结果验证
Sisyphus 收集所有结果，运行 LSP 诊断、构建测试
07
修复 & 完成
发现类型错误 → 用 session_id 继续修复（保持上下文）→ 全部通过 → 报告完成 ✓
